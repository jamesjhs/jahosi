Project

Taskit

Anonymous task-logging PWA · TypeScript / Express 5 / SQLite / vanilla JS

Taskit is a privacy-first, self-hosted task-logging progressive web app built for healthcare staff. It lets users start and stop timed tasks, categorise them, and annotate outcomes — without ever storing a real name, email address, or any patient information. Usernames are auto-generated memorable word pairs, keeping the data useful for workload analysis while remaining fully anonymous.

Over time Taskit builds a rich picture of where time goes — surfacing it through Chart.js analytics, linear regression trendlines, and exportable XLSX reports. An administrator controls the dropdown options, user groups, notices, and messaging from a fully-featured admin panel, while the PWA shell means the app installs on any mobile device and loads instantly.

Task logging — category and subcategory dropdowns, outcome flags

Key features

  • Anonymous by design — usernames are auto-generated memorable word pairs; no real names, emails, or patient data stored
  • PWA — installable on mobile, works offline for cached assets
  • Task tracking — duty vs personal tasks, categories, subcategories, outcomes, and interruption handling
  • Task flags — admin-managed structured annotations (e.g. "Priority too high"); users suggest new flags via email
  • Analytics — session and 30-day history with Chart.js charts, filtering, flag distribution, and linear regression trendlines
  • Excel export — raw task data and full analytics reports downloadable as .xlsx
  • User groups — administrators define which dropdown options each group sees
  • Personal option customisation — users build their own personalised dropdown lists
  • Notices and messages — administrators post notices and send messages to individual users or broadcast to all
  • Admin panel — user management, DB backup/restore, SMTP config, dropdown configuration, registration settings, and group management
  • Configurable registration — three levels for self-registration and invite-based onboarding
  • 30-day data retention — task data automatically deleted after 30 days
  • Health-check endpoint — GET /readyz for uptime monitoring

Security

  • CSRF tokens on all mutating requests
  • bcrypt (cost 12) password hashing; account lockout after repeated failed logins
  • Session ID regeneration on login to prevent session fixation
  • HTTP-only, SameSite=strict session cookies with 30-minute idle timeout and 5-minute inactivity warning
  • Helmet.js security headers; parameterised SQL throughout
  • Automatic HTTPS when Let's Encrypt certificates are present
  • 241-test negative security suite (CSRF, IDOR, SQLi, XSS, input validation, resource exhaustion)
