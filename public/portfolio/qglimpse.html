Project

Qglimpse

Multi-institution visitor insight platform · Kiosk + analytics · Node/Express + React + SQLite

Qglimpse was designed to solve a practical gap: many organisations want regular visitor feedback, but paper forms are inconsistent, expensive to process, and difficult to analyse across locations or time windows. The aim was to make feedback capture simple at the front desk and genuinely useful for operational decisions.

Rationale

Low-friction feedback collection with anonymous, category-based demographic context.

Design

Institution-scoped setup, kiosk mode, flexible question scheduling, and role-aware admin tools.

Implementation

Workspace monorepo with Express API, React PWA shell, SQLite persistence, and SMTP-integrated auth journeys.

Design principles

  • Institution isolation so each organisation manages its own questions, users, and analytics scope.
  • Role-based administration for day-to-day operations across institution and platform levels.
  • Question scheduling by day/time for context-specific data collection windows.
  • Kiosk UX optimised for quick completion in reception and waiting areas.
  • Anonymous feedback model with category-based demographics (no direct personal identifiers).

Implementation highlights

Express API
React + Vite
SQLite (WAL)
Docker-ready

End-administrator workflow focus

Administrators can create institutions, configure kiosk behaviour, manage institution users, tune question sets, and review analytics including cross-tab breakdowns. Operational controls remain focused on delivery and reporting, while security and identity flows support email verification, password reset, optional 2FA, and magic-link access where enabled.
